§01 — About CivicByte

Four standards, six rules, one principle.

Every tool listed on CivicByte has to clear the same four checks before it ships. These aren't suggestions — they're the floor. The six rules underneath them are how we work; the one principle is why.

The four standards.

§02 · Platform · Non-negotiable
Standard 01

Solves a real civic problem.

Every tool must address a specific, documented need identified by or validated with a community. "Interesting dataset" doesn't qualify on its own. We want to know who is stuck and what we're unsticking.

Standard 02

Free and open source.

Freely accessible with no paywalls or sign-ups. Source code must be public under an open license, with data sources cited. If a user has to pay, it won't be listed.

Standard 03

Usable by non-technical users.

Tools are for community members, not developers. Clear interfaces, plain language, mobile-friendly. No CLI, no auth maze, no spreadsheets to download and parse.

Standard 04

Maintained and reliable.

Submitters commit to keeping tools functional and data current. Abandoned tools are archived with a note, not quietly deleted. The platform's value is in things that still work next year.

§03 — The six rules

How we work, day to day.

01

Radical transparency.

Every line of code is public. Every dataset is cited. Civic tech built behind closed doors defeats its own purpose.

02

Community-first design.

Every project starts with a real problem identified by a real community — not by us, not by a dataset.

03

Accessible by default.

If a tool needs technical expertise to use, it has failed. Built for parents, organizers, advocates — not other developers.

04

Open platform.

CivicByte isn't one team's portfolio. It's a shelf where anyone can contribute tools that serve the public good.

05

Impact over portfolio.

A tool used by one PTA is worth more than a demo with a thousand GitHub stars. We count uses, not views.

06

Student-led, not limited.

Being student-founded doesn't mean amateur. Production standards because communities depend on us.

§04 · The principle
Public data should serve the public. The communities that need transparency most often have the least access to it.
— CivicByte, founding charter · 2024

Want to contribute?

CivicByte is open to developers, students, nonprofits, and community organizations. Build something, submit it, report a data gap, or just use the tools and share them with someone who needs them.
